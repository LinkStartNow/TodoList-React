# Specs: README 文档刷新（验收标准）

## 用户故事

- 作为一个新读者，我打开仓库首页能在 30 秒内理解：这是一个什么项目、能做什么、如何跑起来。
- 作为一个开发者，我能复制命令在本地成功启动，并知道常见问题如何处理。

## 验收场景（可执行）

### 场景 A：信息完整且真实

**Given** 我打开仓库首页 README  
**When** 我阅读“功能/技术栈/启动方式/贡献”  
**Then** 内容应与代码事实一致：

- 技术栈必须以 `package.json` 为准（React/Vite/依赖项）
- 不能出现未安装的技术（例如不存在 Tailwind、CSS Modules 等时不能宣称使用）
- 字体、UI 风格描述应与 `index.html` / CSS 实现一致（例如 Google Fonts 引入情况）

### 场景 B：本地运行可复现

**Given** 我在一台全新机器上克隆仓库  
**When** 我按 README 执行

```bash
npm install
npm run dev
```

**Then** 我能在浏览器打开页面并正常使用（添加/完成/删除/编辑 todo）。

### 场景 C：截图/演示占位不误导

**Given** README 包含截图或演示链接字段  
**When** 这些资源尚未提供  
**Then** 必须以“占位/可选”形式呈现，不造成 404 或误导（例如使用占位图片路径并明确说明）。

## 边界条件

- 如果 README 提供部署指南，必须明确它是“可选”且不会阻塞本地运行说明
- README 中所有命令需可在 macOS / Linux 下直接运行（不依赖交互式步骤）
