# Design: README 文档刷新

## 设计原则

- 单一事实来源：以 `package.json`、入口文件与实际代码为准
- 读者优先：先告诉“是什么/能干啥”，再告诉“怎么跑/怎么贡献”
- 最小可维护：结构清晰，后续补截图/部署链接不需要大改

## README 信息架构

1. 项目标题 + 一句话定位
2. 预览（截图占位 + 可选演示链接）
3. 功能列表（只写已实现）
4. 技术栈（以 package.json 为准）
5. 本地运行（最短路径）
6. 常见问题（可选：Node 版本、端口冲突、构建）
7. 贡献方式（PR 流程、代码风格）
8. License（如有）

## 兼容性/约束

- README 命令默认使用 npm（项目已有 package-lock.json）
- 不要求引入 CI、部署或 badge；如添加，应确保不误导且可选
