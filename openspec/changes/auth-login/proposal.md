# Proposal: 前端登录与受保护路由（React Router）

## 背景

当前 TodoList 为单页应用，任何访问都直接进入列表页面。为了支持最基本的访问控制与用户状态，需要引入路由与登录流程。

## 目标

- 引入 React Router，提供：
  - `/login`：登录页
  - `/`：TodoList（受保护路由）
- 未登录访问 `/` 自动跳转到 `/login`
- 登录成功后自动跳回 `/`（或跳回原本想访问的受保护地址）
- 提供退出登录能力
- 登录态存储在 localStorage（或 sessionStorage），并通过 `AuthContext` 统一管理

## 非目标

- 不实现真实后端鉴权（本变更只做前端门禁）
- 不增加多用户 Todo 数据隔离（Todo 仍按当前实现存储）

## 受影响范围

- 前端路由与应用入口
- 新增 AuthContext 及登录页面组件

## 风险与应对

- 刷新/首屏跳转：需要在初始化时从 storage 恢复登录态，并避免闪烁
- 路由升级影响现有逻辑：将 Todo UI 从 `App` 中抽离为页面组件，降低耦合
